# 行程导入功能使用指南

## 功能概述

将 AI 生成的行程规划一键导入至"我的行程"功能已实现。用户可以在 AI 规划页面生成行程后，点击"导入个人行程"按钮，系统会自动解析行程内容并保存到个人行程列表。

## 技术架构

### 后端服务（`trip_import.py`）
- **端口**: 5001
- **功能**: 解析 AI 生成的行程文本，提取结构化数据
- **API 端点**:
  - `POST /api/trip/import` - 解析并导入行程
  - `POST /api/trip/parse` - 仅解析行程（预览用）
  - `GET /api/health` - 健康检查

### 前端集成（`src/services/ai.js`）
- **函数**: `importTripPlan()`
- **功能**: 调用后端解析服务，保存行程到 localStorage
- **UI**: AI 规划页面每条 AI 回复下方显示"导入个人行程"按钮

### 行程解析功能（`parse_trip_plan()`）
自动提取以下信息：
1. **目的地** - 从文本中识别常见目的地
2. **预算** - 提取预算金额（支持千/万单位）
3. **每日行程** - 识别 Day 1、第1天等格式
4. **活动安排** - 提取时间、活动名称
5. **日期范围** - 根据天数自动推算日期

## 使用方法

### 方法一：一键启动（推荐）

1. 双击运行 `start_all_services.bat`
2. 等待所有服务启动（约 5-10 秒）
3. 浏览器会自动打开 `http://localhost:3000`
4. 进入"AI 规划"页面开始使用

### 方法二：手动启动

**步骤 1 - 启动后端 API 服务（端口 5000）**
```bash
python api_server.py
```

**步骤 2 - 启动行程导入 API 服务（端口 5001）**
```bash
python trip_import.py
```

**步骤 3 - 启动前端开发服务器（端口 3000）**
```bash
node dev-server.mjs
```

**步骤 4 - 访问页面**
打开浏览器访问 `http://localhost:3000/ai-plan`

## 操作流程

### 1. 生成行程规划
- 进入"AI 规划"页面
- 输入旅行需求，例如：
  - "帮我规划云南5日游"
  - "春节去日本7天，预算2万"
  - "带父母去北京3天，轻松行程"
- 等待 AI 生成行程规划

### 2. 导入个人行程
- AI 生成回复后，在消息下方会出现"📥 导入个人行程"按钮
- 点击按钮，系统将：
  1. 调用后端解析服务（`localhost:5001`）
  2. 解析行程内容（目的地、日期、每日安排、预算等）
  3. 保存到个人行程列表（localStorage）
  4. 显示成功提示"✅ 行程导入成功！"
  5. 3秒后自动跳转到"行程管理"页面

### 3. 查看导入的行程
- 进入"行程管理"页面（`/trip-manage`）
- 查看导入的行程卡片
- 点击卡片查看详细信息

## 支持格式

### 目的地识别
自动识别常见目的地：
- 国内：北京、上海、广州、深圳、成都、杭州、西安、南京、重庆、武汉、厦门、昆明、大理、丽江、三亚等
- 国际：东京、大阪、京都、首尔、釜山、巴黎、伦敦、纽约、洛杉矶、悉尼等

### 天数识别
支持多种格式：
- `### Day 1: 抵达昆明`
- `## 第1天`
- `**Day 2** - 大理古城`
- `Day3：玉龙雪山`

### 预算识别
支持多种格式：
- `预算：5000元`
- `总预算约8000`
- `¥10000左右`
- `约1万5千元`

## 故障排查

### 问题 1：点击"导入个人行程"无反应
**原因**: 行程导入 API 服务未启动
**解决**:
1. 检查 `trip_import.py` 是否运行
2. 访问 `http://localhost:5001/api/health` 验证服务状态
3. 查看浏览器控制台是否有错误信息

### 问题 2：导入失败提示"网络异常"
**原因**: 后端服务连接失败
**解决**:
1. 确认 `api_server.py` 在端口 5000 运行
2. 确认 `trip_import.py` 在端口 5001 运行
3. 检查防火墙是否阻止端口

### 问题 3：解析后行程信息不完整
**原因**: AI 生成格式不标准
**解决**:
1. 提示 AI 使用标准格式生成行程（Day 1, Day 2...）
2. 手动编辑导入的行程补充信息
3. 在"行程管理"页面点击编辑按钮修改

### 问题 4：前端构建失败
**原因**: 依赖缺失或代码错误
**解决**:
```bash
# 重新安装依赖
rm -rf node_modules
npm install

# 重新构建
npm run build
```
